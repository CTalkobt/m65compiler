# ca45 — Mega65 45GS02 Assembler

`ca45` is a powerful, high-level assembler designed for the MEGA65 (45GS02). It combines traditional assembly with advanced procedural features, dynamic variable management, and multi-pass optimizations.

## Syntax Overview
- **Comments**: 
    - Semicolon (`;`) — Traditional 6502 style.
    - Double-slash (`//`) — C/KickAssembler style (single-line).
    - Slash-star (`/* ... */`) — C/KickAssembler style (multi-line).
- **Labels**: Defined by an identifier followed by a colon (e.g., `name:`).
    - **Scoped Labels**: `ca45` supports scoped labels where colons are used as separators. The `cc45` compiler utilizes this format (e.g., `main:L0:`) to uniquely identify temporary labels within procedures, ensuring they do not collide with labels in other scopes.
    - **Scoping Format**: The `double:colon:` style (the identifier itself containing colons) is utilized for scoping by the C compiler during the assembly phase to flatten the hierarchical scope into globally unique identifiers in the final binary.
    - **Optimization-Transparent Labels**: Labels prefixed with **`@`** (e.g., `@L0:`) are used to define jump targets without breaking the assembler's optimizer. Standard labels act as barriers that reset register tracking; `@` labels allow the optimizer to maintain its knowledge of register states across the label.
- **Equates**: `NAME = value`. Supports arithmetic (`+`, `-`, `*`, `/`) and various literal formats.
- **Local Scoping**: `ca45` supports nested local scopes for labels and variables.
    - **Procedures**: Each `proc` / `endproc` block creates a new named scope.
    - **Anonymous Blocks**: Curly braces `{ ... }` create anonymous local scopes for isolating labels within a procedure or global area.
    - **Structured Data Referencing**: By combining labels and anonymous blocks, you can create struct-like data structures that are accessible via colon-separated paths (e.g., `outer:inner:element`).
        ```assembly
        mystruct:
        {
          a: .byte $00
          b: .word $0000
        }

        lda mystruct:a    ; Accesses the byte at the start of mystruct
        ldax mystruct:b   ; Accesses the word at offset 1
        ```
    - **Resolution**: Symbols are resolved using a hierarchical lookup (Inner Scope -> Parent Scope -> Global Scope). This allows reuse of common label names (like `loop:`) without conflict.

- **Case Sensitivity**: Mnemonics and register names are case-insensitive.

## Literal Formats
- **Hexadecimal**: `$1234`
- **Decimal**: `123`
- **Binary**: `%1010`
- **Character**: `'A'` (converted to PETSCII in `.text` context)
- **String**: `"Hello"` (PETSCII or ASCII depending on directive)

## Command-Line Usage
```bash
ca45 [options] <input_file.s>
```

### Options
- `-o <file>`: Specify the output binary file name (default is `out.bin`).
- `-v`: Verbose mode.
- `-Dname=val`: Define a symbol. Special symbols:
    - `-Dcc45.zeroPageStart=$addr`: Set the base address for stack-relative and indirect operations (default: `$02`).
- `-?`: Display help and exit.

## Supported Opcodes
For a complete list of supported instructions and addressing modes, see the [45GS02 Instruction Reference](opcodes.md).

`ca45` supports the full 45GS02 instruction set, including standard 6502/65CE02 instructions and MEGA65 enhancements:
- **Z Register**: `ldz`, `stz`, `inz`, `dez`, `phz`, `plz`.
- **Word Instructions**: `phw`, `inw`, `dew`, `asw`, `row`, `bsr`.
- **Extended Branches**: 16-bit relative branches (`bsr`, `bra` relfar).
- **Stack-Relative Addressing**: `lda offset, s`.
- **Flat Indirect Addressing**: `lda [zp],z`. Supports `adc`, `and`, `cmp`, `eor`, `lda`, `ora`, `sbc`, and `sta`. Automatically emits the `eom` (`0xEA`) prefix.
- **Instruction Prefix**: `eom` (`0xEA`) is supported as a standalone mnemonic.
- **Reserved Mnemonics**: The literal `nop` mnemonic is **disallowed** in `ca45` as `$EA` is now an instruction prefix on the 45GS02. Use `eom` if you specifically want the `$EA` byte, or `clv` (`$B8`) as a safe 1-byte NOP alternative.

## Simulated Opcodes (Mega65 Hardware Assisted)
`ca45` provides high-level simulated opcodes that leverage the MEGA65's hardware acceleration or provide convenient multi-instruction sequences.

### Legend
- **Registers**: `.A`, `.X`, `.Y`, `.Z` (8-bit), `.AX`, `.AY`, `.AZ`, `.XY` (16-bit), `.Q` (32-bit).
- **Flags**: `N` (Negative), `Z` (Zero), `C` (Carry), `V` (Overflow).

---

### `mul.<width> <dest>, <source>`
Performs hardware-accelerated multiplication (`dest = dest * source`).
- **Widths**: `.8`, `.16`, `.24`, `.32` bits.
- **Modes**: 
    - `dest`: Register pair (`.AX`, `.AXY`, `.Q`) or `abs` memory.
    - `source`: `#imm`, `abs`, register pair, or `offset, s`.
- **Registers Affected**: `.A` (used for transfer), `dest` registers.
- **Flags Affected**: `N`, `Z` (based on last byte stored to RAM or last register loaded).

### `div.<width> <dest>, <source>`
Performs hardware-accelerated division (`dest = dest / source`).
- **Widths**: `.8`, `.16`, `.24`, `.32` bits.
- **Modes**: same as `mul`.
- **Registers Affected**: `.A`, `dest` registers.
- **Flags Affected**: `N`, `Z`.

### `ldax / lday / ldaz <src>`
Loads a 16-bit word into a register pair.
- **Modes**: `#imm`, `abs`, `offset, s`.
- **Registers Affected**: `dest` pair (e.g., `.A` and `.X`).
- **Flags Affected**: `N`, `Z` (based on high byte).

### `ldx / ldy / ldz <offset>, s`
Loads an 8-bit value from the stack into a register.
- **Modes**: `offset, s`.
- **Generated Code**: Uses `lda <offset>, s` followed by `tax/tay/taz`.
- **Registers Affected**: `dest` register, `.A` (clobbered).
- **Flags Affected**: `N`, `Z`.

### `stax / stay / staz <dest>`
Stores a 16-bit word from A (low) and X/Y/Z (high) into memory.
- **Modes**: `abs`, `offset, s`.
- **Registers Affected**: `.A` (if `dest` is `offset, s`).
- **Flags Affected**: `N`, `Z` (if `dest` is `offset, s`).

### `stx / sty / stz <offset>, s`
Stores an 8-bit value from a register into the stack.
- **Modes**: `offset, s`.
- **Generated Code**: Uses `pha`, `txa/tya/tza`, `sta <offset>, s`, `pla`.
- **Registers Affected**: `.A` (preserved).
- **Flags Affected**: None (preserved).

### `add.16 / sub.16 / add.s16 / sub.s16 <dest>, <src>`
16-bit addition/subtraction.
- **Modes**:
    - `dest`: `.AX`, `.AY`, `.AZ` or `abs`.
    - `src`: `#imm`, `abs`, or register pair.
- **Registers Affected**: `dest` registers, `.A` (temp).
- **Flags Affected**: `N`, `Z`, `C`, `V`.

### `and.16 / ora.16 / eor.16 .ax, <src>`
16-bit bitwise logic.
- **Modes**: `#imm`, `abs`.
- **Registers Affected**: `.AX`, `.A` (temp).
- **Flags Affected**: `N`, `Z`.

### `lsl.16 / lsr.16 / lsl.s16 / lsr.s16 [dest]`
16-bit logical shifts.
- **Modes**: `.ax`, `abs`.
- **Registers Affected**: `dest` registers.
- **Flags Affected**: `N`, `Z`, `C`.

### `asr.16 / asr.s16 [dest]`
16-bit arithmetic right shift (preserves sign).
- **Modes**: `.ax`, `abs`.
- **Registers Affected**: `.A` (temp), `dest` registers.
- **Flags Affected**: `N`, `Z`, `C`.

### `neg.16 / not.16 / neg.s16 [dest]`
16-bit negation or bitwise NOT.
- **Modes**: `.ax`, `abs`, `offset, s`.
- **Registers Affected**: `dest` registers, `.A`, `.X` (if stack).
- **Flags Affected**: `N`, `Z`, `C` (if `neg`).

### `cmp.16 / cmp.s16 .ax, <src>`
16-bit comparison (unsigned or signed).
- **Modes**: `#imm`, `abs`.
- **Registers Affected**: `.A` (temp).
- **Flags Affected**: `N`, `Z`, `C`.

### `abs.16 / abs.s16 [dest]`
16-bit absolute value.
- **Modes**: `.ax`, `abs`, `offset, s`.
- **Registers Affected**: `dest` registers, `.A`, `.X` (if stack).
- **Flags Affected**: `N`, `Z`, `C`.

### `rol.16 / ror.16 / rol.s16 / ror.s16 [dest]`
16-bit rotate through carry.
- **Modes**: `.ax`, `abs`.
- **Registers Affected**: `dest` registers.
- **Flags Affected**: `N`, `Z`, `C`.

### `chkzero.8 / chkzero.16 / chknonzero.8 / chknonzero.16`
Zero-detection boolean result.
- **Registers Affected**: `.A` (contains 1 or 0), `.X` (if 16-bit).
- **Flags Affected**: `N`, `Z`.

### `branch.16 <condition>, <target>`
16-bit branching (beq/bne).
- **Registers Affected**: `.X` (temp).
- **Flags Affected**: None (preserves incoming flags except via internal CMP).

### `select <reg>, <val_if_true>, <val_if_false>`
Conditional assignment based on Zero flag.
- **Registers Affected**: `reg`.
- **Flags Affected**: `N`, `Z`.

### `ptrstack <offset>`
Load address of stack variable into `.AX`.
- **Registers Affected**: `.AX`.
- **Flags Affected**: `N`, `Z`, `C`.

### `ptrderef <zp_addr>`
De-reference 16-bit pointer in Zero Page into `.AX`.
- **Registers Affected**: `.AX`, `.Y`.
- **Flags Affected**: `N`, `Z`.

### `ldw.f / stw.f <addr28>`
28-bit linear memory 16-bit load/store.
- **Registers Affected**: `.AX`.
- **Flags Affected**: `N`, `Z`.

### `ldw.sp / stw.sp <src/dest>, <offset>`
Preferred stack-relative load/store.
- **Registers Affected**: `.AX`, `.Y` (if `.AY`), `.Z` (if `.AZ`).
- **Flags Affected**: `N`, `Z`.

### `fill / fill.sp <dest>, <length>`
DMA Memory Fill.
- **Value**: taken from `.A`.
- **Modes**: `abs`, `(zp)`, register pair, `offset, s`.
- **Registers Affected**: None (preserved).
- **Flags Affected**: None.

### `move / move.sp <src>, <dest>`
DMA Memory Copy.
- **Length**: taken from `.XY`.
- **Modes**: `abs`, `.AZ`, `offset, s`.
- **Registers Affected**: None.
- **Flags Affected**: None.

### `inc.f / dec.f <addr28>`
28-bit linear memory byte inc/dec.
- **Flags Affected**: `N`, `Z`.

### `swap <reg1>, <reg2>`
Swap two 8-bit registers (`A`, `X`, `Y`, `Z`).
- **Registers Affected**: `reg1`, `reg2`.
- **Flags Affected**: None (if using stack).

### `push <reg> / pop <reg>`
Multi-register push/pop.
- **Registers**: any combination or width.
- **Flags Affected**: None (pop preserves P).

### `zero <reg1>[, ...]`
Clear registers to zero.
- **Registers Affected**: specified registers.
- **Flags Affected**: `Z` (1), `N` (0).

### `sxt.8`
Sign-extend `.A` to `.AX`.
- **Registers Affected**: `.X`.
- **Flags Affected**: `N`, `Z` (based on X).

### `expr <target>, <expression>`
Complex math expression.
- **Registers Affected**: `target`, plus stack usage.
- **Flags Affected**: `N`, `Z`, `C`, `V`.

## Procedures and Stack Management
`ca45` provides directives to manage 45GS02-style procedures with automated stack framing:

### `proc <name>[, [b#|w#]<param1>, ...]`
Defines a procedure and its named parameters.
- **Parameters**: Can be prefixed with `b#` (8-bit byte) or `w#` (16-bit word, default).
- **Mapping**: Parameters are automatically mapped to the correct stack offset. `arg1`, `arg2`, etc., are also available as positional aliases.

### `call <name>[, <arg1>, ...]`
Invokes a procedure with automated parameter passing.
- **Arguments**: Pushed onto the stack before the `jsr`. Supports immediate values (`#$10`), labels, or existing stack variables.
- **Inference**: Size of arguments is inferred from the symbol table or forced with `b#`/`w#`.

### `endproc`
Completes a procedure block.
- **Cleanup**: Automatically calculates the total size of all parameters and emits an optimized `rtn #n` or `rts` to clean the stack and return to the caller.

## Directives (Pseudo-ops)
- `.cpu <name>`: Selects the target CPU (e.g., `.cpu _45gs02`). Defaults to 45GS02 if unspecified or unsupported.
- `.org <addr>`: Sets the current Program Counter (PC).
- `* = <addr>`: Alternative syntax for `.org` (KickAssembler compatibility).
- `.byte <v1>[, ...]` : Emits 8-bit bytes.
- `.word <v1>[, ...]` : Emits 16-bit words (little-endian).
- `.dword <v1>[, ...]` : Emits 32-bit double words (little-endian).
- `.long <v1>[, ...]` : Alias for `.dword`.
- .float <v1>[, ...]` : Emits 40-bit (5-byte) Commodore floating point values.
- `.res <n>[, <fill>]`: Reserves `<n>` bytes of memory. If in a data or code segment, these bytes are initialized with `<fill>` (defaults to `$00`). If in a `.bss` segment, no data is emitted, but the program counter is advanced.
- `.align <n>[, <fill>]`: Aligns the program counter to the next multiple of `<n>`. An optional `<fill>` byte can be specified (default is `$00`).
- `.balign <n>[, <fill>]`: Alias for `.align`.
- `.segment "<name>"`: Switches the active segment. Subsequent code and data will be placed in this segment.
- `.segmentOrder <name1>[, <name2>, ...]`: Defines the order in which segments are emitted to the final binary. Default is `code, data, bss`.
    - **Note on Ordering**: Segments must be address-monotonic. If a segment's start address (defined by `.org`) is lower than the current emission pointer from previous segments, an error is thrown.
    - **Gap Filling**: Any gaps between segments with explicit `.org` addresses are automatically 0-filled in binary mode.
    - **Base Address**: The first `.org` encountered in the first emitted segment defines the base address of the binary without adding leading zeros.
- `.code`: Shortcut for `.segment "code"`.
- `.text`: Shortcut for `.segment "code"` (for compatibility).
- `.data`: Shortcut for `.segment "data"`.
- `.bss`: Shortcut for `.segment "bss"`. BSS segments are used for variable reservations and do not produce output in the final binary.
- `.segment "<name>" { ... }`: Temporarily switches to the specified segment for the duration of the block.
- `.text "<string>"` : Emits a PETSCII-encoded string (case-swapped).
- `.ascii "<string>"`: Emits a raw ASCII-encoded string.
- `.basicUpstart <address>` : Emits a C64 BASIC stub (`10 SYS <address>`) to start a machine language program.
- `.var <name> = <expr>`: Defines or updates an assembler-time variable.
- `.cleanup <n>`: Adjusts the internal stack tracking for the current procedure.

## Register and Flag References
`ca45` allows referencing CPU registers and status flags directly within expressions:
- **8-bit Registers**: `.a`, `.x`, `.y`, `.z`, `.sp` (low byte).
- **16-bit Registers**: `.ax` (a=low, x=high), `.ay` (a=low, y=high), `.az` (a=low, z=high), `.xy` (x=low, y=high).
- **Special Registers**: `.pc` (Current Program Counter).
- **32-bit Q-Register**: `.q` or `.axyz` (a=byte0, x=byte1, y=byte2, z=byte3).
- **Processor Status Flags**: `p.c` (Carry), `p.z` (Zero), `p.i` (Interrupt), `p.d` (Decimal), `p.b` (Break), `p.v` (Overflow), `p.n` (Negative), `p.e` (Emulation).

## Optimizations
`ca45` uses a multi-pass approach to optimize the final binary:
- **Tiered Branch Strategy**: All branch instructions (`bra`, `beq`, `bne`, `bcc`, `bcs`, `bpl`, `bmi`, `bvc`, `bvs`) automatically select the most efficient addressing mode (8-bit relative or 16-bit relative) based on the target offset. The assembler uses a multi-pass approach to recalculate instruction sizes and addresses until they stabilize, ensuring the most compact representation without requiring manual padding.
- **JMP to BRA Conversion**: `jmp label` is automatically converted to `bra label` (8-bit or 16-bit) to enable relative shortening.
- **Redundant Load Elimination**: The assembler identifies and removes redundant `lda`, `ldx`, `ldy`, and `ldz` instructions. For example, a load immediately following a store to the same address is eliminated. Identical immediate loads are also removed.
- **Local Optimization Windows**: Labels prefixed with `@` are treated as internal targets that do not break register tracking, allowing for deeper optimization across these points.
- **Dead Code Removal**: Instructions appearing after a return (`rts`, `rtn`, `rti`) within a procedure block are automatically identified and removed from the final binary until the next label is encountered.
- **Return Optimization**: `rtn #0` is automatically converted to the single-byte `rts` (`0x60`) instruction.

## Implementation Details

### Major Structures
- **`AssemblerLexer`**: Converts assembly source into a stream of tokens. Supports semicolon, `//`, and `/* */` comments.
- **`AssemblerParser`**: The heart of the assembler. It performs two passes over the tokens:
    - **Pass 1**: Identifies all labels, determines the size of each instruction and directive, and builds the symbol table.
    - **Pass 2**: Resolves all expressions and symbol references, and emits the final machine code bytes.
- **`M65Emitter`**: A low-level utility class used during Pass 2 to emit specific instruction bytes based on the MEGA65 architecture.
- **`SymbolTable`**: A hierarchical map of symbol names to `Symbol` objects, supporting nested scoping.

### Register and Flag Tracking
- **Registers**: The assembler identifies and removes redundant `lda`, `ldx`, `ldy`, and `ldz` instructions within local optimization windows (between non-`@` labels). However, it does not track register state for the purpose of simplifying high-level `expr` or simulated opcodes.
- **Flags**: Processor flags are not tracked. However, flags can be used as values in the `EXPR` simulated opcode (e.g., `EXPR .A, P.C`).

### Adding Custom Opcodes
To add or modify opcodes in the assembler:
1.  **Opcode Map**: Add a new entry to the static `opcodes` map in `src/main/AssemblerParser.cpp` (within the `getOpcodeMapInternal()` function). You must specify the mnemonic, the `AddressingMode`, and the hexadecimal opcode.
2.  **Addressing Modes**: If a new addressing mode is needed, add it to the `AddressingMode` enum in `include/AssemblerParser.hpp` and update the `calculateInstructionSize()` and Pass 2 logic in `AssemblerParser.cpp`.

### Adding New Pseudo-ops (Directives)
To add a new directive (e.g., `.my_directive`):
1.  **Lexer**: The lexer automatically identifies tokens starting with `.` as directives.
2.  **Pass 1**: Add a handler for the directive in `AssemblerParser::pass1()` (within the `match(AssemblerTokenType::DIRECTIVE)` block). This is where you determine if the directive has a size (e.g., `.byte` has a size, while `.var` does not).
3.  **Pass 2**: Add a corresponding handler in `AssemblerParser::pass2()` to emit the actual data or perform the action during the final code generation.
