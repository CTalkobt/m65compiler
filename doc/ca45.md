# ca45 — Mega65 45GS02 Assembler

`ca45` is a powerful, high-level assembler designed for the MEGA65 (45GS02). It combines traditional assembly with advanced procedural features, dynamic variable management, and multi-pass optimizations.

## Syntax Overview
- **Comments**: 
    - Semicolon (`;`) — Traditional 6502 style.
    - Double-slash (`//`) — C/KickAssembler style (single-line).
    - Slash-star (`/* ... */`) — C/KickAssembler style (multi-line).
- **Labels**: Defined by an identifier followed by a colon (e.g., `name:`).
- **Equates**: `NAME = value`. Supports arithmetic (`+`, `-`, `*`, `/`) and various literal formats.
- **Local Scoping**: `ca45` supports nested local scopes for labels and variables.
    - **Procedures**: Each `proc` / `endproc` block creates a new named scope.
    - **Anonymous Blocks**: Curly braces `{ ... }` create anonymous local scopes for isolating labels within a procedure or global area.
    - **Resolution**: Symbols are resolved using a hierarchical lookup (Inner Scope -> Parent Scope -> Global Scope). This allows reuse of common label names (like `loop:`) without conflict.
- **Case Sensitivity**: Mnemonics and register names are case-insensitive.

## Literal Formats
- **Hexadecimal**: `$1234`
- **Decimal**: `123`
- **Binary**: `%1010`
- **Character**: `'A'` (converted to PETSCII in `.text` context)
- **String**: `"Hello"` (PETSCII or ASCII depending on directive)

## Supported Opcodes
`ca45` supports the full 45GS02 instruction set, including standard 6502/65CE02 instructions and MEGA65 enhancements:
- **Z Register**: `LDZ`, `STZ`, `INZ`, `DEZ`, `PHZ`, `PLZ`.
- **Word Instructions**: `PHW`, `INW`, `DEW`, `ASW`, `ROW`, `BSR`.
- **Extended Branches**: 16-bit relative branches (`BSR`, `BRA` relfar).
- **Stack-Relative Addressing**: `LDA offset, s`.
- **Flat Indirect Addressing**: `LDA [zp],Z`. Supports `ADC`, `AND`, `CMP`, `EOR`, `LDA`, `ORA`, `SBC`, and `STA`. Automatically emits the `EOM` (`0xEA`) prefix.

## Simulated Opcodes (Mega65 Hardware Assisted)
`ca45` provides high-level simulated opcodes that leverage the MEGA65's hardware acceleration or provide convenient multi-instruction sequences:

### `mul.<width> <dest>, <source>`
...
### `inc.w <offset>, s` / `dec.w <offset>, s`
Performs a 16-bit increment or decrement of a value on the stack at the specified offset.
- **Generated Code**: Uses `TSX` and `INC $0101+offset,X` (for increment) or `LDA / DEC` (for decrement) to perform a thread-safe stack-relative word operation. This is especially useful for manipulating struct members stored on the stack.
- **Mnemonics**: Both `INW offset, s` and `DEW offset, s` are supported as aliases for these simulated operations when the operand is stack-relative.
Performs a hardware-accelerated multiplication (`dest = dest * source`).
- **`<width>`**: `.8`, `.16`, `.24`, or `.32` bits.
- **`<dest>`**: 
    - Register prefix: `.A` (for `.8`), `.AX` (for `.16`), `.AXY` (for `.24`), `.AXYZ` or `Q` (for `.32`).
    - Memory address (e.g., `$1234`).
- **`<source>`**: Constant expression, address, register, or stack-relative (e.g., `2, s`).
- **Generated Code**: Automatically loads `MULTINA` and `MULTINB` registers, then stores the result from `MULTOUT` back to the destination.

### `div.<width> <dest>, <source>`
Performs a hardware-accelerated division (`dest = dest / source`).
- **`<width>`**: `.8`, `.16`, `.24`, or `.32` bits.
- **`<dest>`**: Same options as `mul`.
- **`<source>`**: Same options as `mul`.
- **Generated Code**: Loads math registers, waits for `DIVBUSY` to clear, and stores the quotient from `DIVOUT` back to the destination.
- **Division by Zero**: 
    - **Assembly Time**: Constant division by zero (e.g., `div.8 .A, 0` or `expr .A, 10/0`) results in an **assembly-time error**.
    - **Runtime**: Division by zero where the divisor is a register or memory location is **undefined**. The hardware state and resulting destination value are non-deterministic.

### `expr <target>, <expression>`
Evaluates a high-level mathematical expression and generates the corresponding machine code to store the result in the target.
- **`<target>`**: 
    - **Register**: Results are moved to the specified register (e.g., `.A`, `.AX`, `.Q`).
    - **Memory**: If target is a symbol or address, code is generated to store the result in that RAM location (e.g., `expr my_var, 3 + 5` emits `STA $addr`).
- **`<expression>`**: Mathematical formula supporting `+`, `-`, `*`, `/`, and parentheses.
- **Advanced Features**:
    - **Recursion**: Supports complex nested formulas (e.g., `expr .A, (.X + 5) * (.Y - 2)`). The engine uses the CPU stack for intermediate results.
    - **Constant Folding**: Fully constant sub-expressions are evaluated at assembly time.
    - **Hardware Acceleration**: Multiplication and Division automatically use the MEGA65 hardware math unit.
    - **Flag Capture**: Status flags (`P.C`, `P.Z`, etc.) can be used as values (0 or 1). `P.C` is highly optimized.
- **Division by Zero**: 
    - **Assembly Time**: Constant division by zero results in an **assembly-time error**.
    - **Runtime**: Division by zero is **undefined**.

- `.var <name> = <expr>`: Defines or updates an **assembler-time** variable. This updates the assembler's internal symbol table but **does not generate machine code**. Use this for address pointers or loop counters used by the assembler.

           | <identifier>

<expression> ::= <term> ( ("+" | "-") <term> )*
<term>       ::= <primary> ( ("*" | "/") <primary> )*

<flag-ref>     ::= "P.C" | "P.Z" | "P.I" | "P.D" | "P.B" | "P.V" | "P.N" | "P.E"

### `proc <name>[, [B#|W#]<param1>, ...]`
Defines a procedure and its named parameters.
- **Parameters**: Can be prefixed with `B#` (8-bit byte) or `W#` (16-bit word, default).
- **Mapping**: Parameters are automatically mapped to the correct stack offset. `ARG1`, `ARG2`, etc., are also available as positional aliases.

### `call <name>[, <arg1>, ...]`
Invokes a procedure with automated parameter passing.
- **Arguments**: Pushed onto the stack before the `JSR`. Supports immediate values (`#$10`), labels, or existing stack variables.
- **Inference**: Size of arguments is inferred from the symbol table or forced with `B#`/`W#`.

### `endproc`
Completes a procedure block.
- **Cleanup**: Automatically calculates the total size of all parameters and emits an optimized `RTN #n` or `RTS` to clean the stack and return to the caller.

## Directives (Pseudo-ops)
- `.cpu <name>`: Selects the target CPU (e.g., `.cpu _45gs02`). Defaults to 45GS02 if unspecified or unsupported.
- `.org <addr>`: Sets the current Program Counter (PC).
- `* = <addr>`: Alternative syntax for `.org` (KickAssembler compatibility).
- `.byte <v1>[, ...]` : Emits 8-bit bytes.
- `.word <v1>[, ...]` : Emits 16-bit words (little-endian).
- `.dword <v1>[, ...]` : Emits 32-bit double words (little-endian).
- `.long <v1>[, ...]` : Alias for `.dword`.
- `.float <v1>[, ...]` : Emits 40-bit (5-byte) Commodore floating point values.
- `.text "<string>"` : Emits a PETSCII-encoded string (case-swapped).
- `.ascii "<string>"`: Emits a raw ASCII-encoded string.
- `.basicUpstart <address>`: Emits a C64 BASIC stub (`10 SYS <address>`) to start a machine language program.
- `.var <name> = <expr>`: Defines or updates an assembler-time variable.
- `.cleanup <n>`: Adjusts the internal stack tracking for the current procedure.

## Register and Flag References
`ca45` allows referencing CPU registers and status flags directly within expressions:
- **8-bit Registers**: `.A`, `.X`, `.Y`, `.Z`, `.SP` (low byte).
- **16-bit Registers**: `.AX` (A=low, X=high), `.AY` (A=low, Y=high), `.AZ` (A=low, Z=high), `.XY` (X=low, Y=high).
- **32-bit Q-Register**: `.Q` or `.AXYZ` (A=byte0, X=byte1, Y=byte2, Z=byte3).
- **Processor Status Flags**: `P.C` (Carry), `P.Z` (Zero), `P.I` (Interrupt), `P.D` (Decimal), `P.B` (Break), `P.V` (Overflow), `P.N` (Negative), `P.E` (Emulation).

### Floating Point Format (.float)
The `.float` directive emits numbers in the standard Commodore 6502/45GS02 5-byte format:
- **Byte 0**: Exponent (Excess 128).
- **Byte 1**: Mantissa Byte 1 (Sign bit in Bit 7, hidden bit assumed for non-zero).
- **Bytes 2-4**: Mantissa Bytes 2-4.

## Optimizations
`ca45` uses a multi-pass approach to optimize the final binary:
- **Tiered Branch Strategy**: All branch instructions (`BRA`, `BEQ`, `BNE`, `BCC`, `BCS`, `BPL`, `BMI`, `BVC`, `BVS`) automatically select the most efficient addressing mode (8-bit relative or 16-bit relative) based on the target offset. If a branch is optimized to 8-bit from a 16-bit reservation, a NOP/EOM (`0xEA`) is used to maintain address alignment.
- **Dead Code Removal**: Instructions appearing after a return (`RTS`, `RTN`, `RTI`) within a procedure block are automatically identified and removed from the final binary until the next label is encountered.
- **Return Optimization**: `RTN #0` is automatically converted to the single-byte `RTS` (`0x60`) instruction.

## Implementation Details
- **Pass 1**: Collects symbols, calculates sizes (branch instructions default to 3 bytes), and maps procedure contexts.
- **Pass 2**: Emits binary encoding, performs branch optimization, and removes dead code.
