# cp45 — Mega65 C Preprocessor

The preprocessor is a core component of the `cc45` suite. It is integrated directly into the `cc45` compiler and `ca45` assembler pipelines as their first pass, but can also be used as a standalone tool (`cp45`).

## Usage

By default, `cp45` (or `cc45 -E`) outputs the preprocessed source to `stdout`. Use the `-o <file>` option to redirect output to a file.

- **Standalone**: `cp45 [options] input_file`
- **Via Compiler**: `cc45 -E [options] input_file.c` (stops after preprocessing)

## Supported Directives

### `#include`
Supports `#include "filename"`, `#include <filename>`, and `#include MACRO`.
- **Search Path**:
  1. The directory containing the current file.
  2. Any include paths provided via command-line arguments (e.g., `-Ipath`).
- **Macro Expansion**: If the target is not a literal string or angle-bracketed name, it is expanded as a macro before inclusion.
- **Recursion**: Supported up to a reasonable depth (e.g., 16 levels) to prevent circular inclusions.

### `#pragma include_once`
If placed on the first non-whitespace line of a file, ensures the file is only included once. Subsequent include attempts for the same file will be silently ignored.

### `#define SYMBOL [value]`
Defines a preprocessor symbol.
- **Function-like Macros**: Supported using `SYMBOL(params)`.
- **Variadic Macros**: Supported using `SYMBOL(params, ...)` and the `__VA_ARGS__` identifier in the body.
- **Comma Removal**: Supports the GNU/Clang extension where `##__VA_ARGS__` removes the preceding comma if no variable arguments are provided.
- **Macro Operators**:
    - `#`: Stringification (converts a macro parameter to a string literal).
    - `##`: Token pasting (concatenates two tokens).
- **Scope**: Symbols defined in the preprocessor are separate from C variables or Assembler labels.

### `_Pragma("string")`
Allows generating pragmas from within macro expansions.

## Standard Predefined Macros
The following macros are automatically expanded in active code:
- `__FILE__`: The current filename (as a string literal).
- `__LINE__`: The current line number (as an integer).
- `__DATE__`: The compilation date (as a string literal).
- `__TIME__`: The compilation time (as a string literal).
- `__STDC__`, `__STDC_VERSION__`, `__STDC_HOSTED__`: Standard C version and hosting identifiers.

## Compiler Pass-through (`#!`)

To allow the C compiler to generate assembly that contains its own preprocessor directives (for the assembler), the `#!` prefix is supported. 

When the `cc45` compiler's preprocessor encounters a line starting with `#!`, it does **not** process it. Instead, it strips the `!` and emits the resulting `#directive` directly into the assembly output.

**Example C Code**:
```c
#!ifdef _USE_HARDWARE_MUL_
    result = a * b;
#!else
    result = software_mul(a, b);
#!endif
```

**Generated Assembly**:
```asm
#ifdef _USE_HARDWARE_MUL_
    ; ... generated assembly for result = a * b ...
#else
    ; ... generated assembly for software_mul ...
#endif
```
