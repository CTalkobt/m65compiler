# cp45 — Mega65 C Preprocessor

The preprocessor can be invoked in two ways:
1.  **Directly**: Using the `cp45` binary.
2.  **Via Compiler**: Using the `cc45 -E` flag.

By default, `cp45` (and `cc45 -E`) outputs the preprocessed source to `stdout`. Use the `-o <file>` option to redirect output to a file.

## Overview

The preprocessor provides a unified mechanism for conditional compilation, file inclusion, and macro definitions. While both tools share the same preprocessor logic, they operate independently on their respective source files. A special pass-through syntax (`#!`) allows the compiler to emit preprocessor directives directly into the generated assembly for later processing by the assembler.

## Supported Directives

### `#include "filename"`
- **Behavior**: Replaces the directive line with the entire contents of the specified file.
- **Search Path**:
  1. The directory containing the current file.
  2. Any include paths provided via command-line arguments (e.g., `-Ipath`).
- **Recursion**: Supported up to a reasonable depth (e.g., 16 levels) to prevent circular inclusions.

### `#define SYMBOL [value]`
- **Behavior**: Defines a preprocessor symbol.
- **Function-like Macros**: Supported using `SYMBOL(params)`.
- **Variadic Macros**: Supported using `SYMBOL(params, ...)` and the `__VA_ARGS__` identifier in the body.
- **Comma Removal**: The `##__VA_ARGS__` sequence is supported to remove the preceding comma if no variable arguments are provided.
- **Macro Operators**:
    - `#`: Stringification (converts a macro parameter to a string literal).
    - `##`: Token pasting (concatenates two tokens).
- **Scope**: Symbols defined in the preprocessor are separate from C variables or Assembler labels.

### `_Pragma("string")`
- **Behavior**: The `_Pragma` operator allows generating pragmas from within macro expansions.

### `#include "filename"` / `#include <filename>` / `#include MACRO`
- **Behavior**: Includes the contents of the specified file.
- **Macro Expansion**: If the target is not a literal string or angle-bracketed name, it is expanded as a macro.

### `#pragma include_once`
- **Behavior**: If placed on the first non-whitespace line of a file, ensures the file is only included once. Subsequent include attempts for the same file will be silently ignored.

## Standard Predefined Macros
The following macros are automatically expanded in active code:
- `__FILE__`: The current filename (as a string literal).
- `__LINE__`: The current line number (as an integer).
- `__DATE__`: The compilation date (as a string literal).
- `__TIME__`: The compilation time (as a string literal).

## Compiler Pass-through (`#!`)

To allow the C compiler to generate assembly that contains its own preprocessor directives (for the assembler), the `#!` prefix is introduced.

### `#!directive`
- **Behavior**: When the `cc45` compiler encounters a line starting with `#!`, it does **not** process it as a C preprocessor directive. Instead, it strips the `!` and emits the resulting `#directive` directly into the assembly output at the current position.
- **Example**:
  ```c
  #!ifdef _USE_HARDWARE_MUL_
    // This C code will only be active in the final binary if the ASSEMBLER 
    // sees _USE_HARDWARE_MUL_ defined.
    result = a * b;
  #!else
    result = software_mul(a, b);
  #!endif
  ```
- **Generated Assembly**:
  ```asm
  #ifdef _USE_HARDWARE_MUL_
    ; ... generated assembly for result = a * b ...
  #else
    ; ... generated assembly for software_mul ...
  #endif
  ```

## Implementation Plan

### 1. Shared Preprocessor Component
- Implement a `Preprocessor` class in `src/main/Preprocessor.cpp` and `include/Preprocessor.hpp`.
- The `process()` method will perform a single pass (or recursive passes) over the source string.
- It will handle `#include` by reading files and injecting their content.
- It will handle `#ifdef`/`#else`/`#endif` by stripping or keeping lines based on symbol definitions.
- It will handle `#!` by transforming them into a format that the compiler's parser can recognize as a "top-level assembly directive" or "inline assembly".

### 2. Compiler Integration (`cc45`)
- **Main Loop**: Update `src/main/cc45_main.cpp` to invoke the `Preprocessor` before the `Lexer`.
- **Pass-through**: The preprocessor will transform `#!ifdef SYMBOL` into `__asm__("#ifdef SYMBOL");`.
- **Parser Update**: Modify `Parser::parse()` in `src/main/Parser.cpp` to allow `ASM` statements at the top level of a `TranslationUnit`. Currently, `ASM` is only expected inside function bodies.
- **Code Generator**: The `CodeGenerator` will emit these top-level `ASM` strings directly into the output `.s` file.

### 3. Assembler Integration (`ca45`)
- **Main Loop**: Update `src/main/ca45_main.cpp` to invoke the `Preprocessor` before the `AssemblerLexer`.
- **Directives**: The assembler will process `#include`, `#define`, etc., normally. Since its output is binary, it does not support the `#!` pass-through (it would be ignored or treated as a regular `#` if the assembler was part of a longer chain).

### 4. Command-Line Arguments
- Support `-I<path>` for specifying include directories.
- Ensure `-D<symbol>` flags are passed to the preprocessor's initial symbol table.

## Constraints and Requirements
- **File Encoding**: All included files are assumed to be in the same encoding as the primary source (PETSCII/ASCII).
- **Line Numbers**: The preprocessor maintains line number information by injecting empty lines for stripped or directive lines, ensuring that compiler/assembler errors point to the correct locations in the original source files.
